    XLENT Compliance-scanner

    Beskytt sensitive kundedata — sjekk dokumentet før du limer inn i ChatGPT, Claude eller Copilot.

    Hva gjør verktøyet?

    Compliance-scanner analyserer dokumenter automatisk og leter etter sensitiv informasjon som personnumre, navn, API-nøkler, kommersielt sensitive tall (timespriser, dagspriser, prosjektsummer, marginer og rabatter) og konfidensielle nøkkelord. Funnene presenteres som et trafikklysnivå slik at du raskt ser om dokumentet er trygt å dele.

    🔒100 % lokal kjøring — ingen dokumenter, tekst eller funn sendes over internett.
    Filen forlater aldri maskinen din.

    Hva søker verktøyet etter?

    • Personnumre og fødselsdatoer (norsk mod-11-validering)
    • Personnavn — via maskinlæring (spaCy NER, lastes automatisk)
    • E-postadresser og telefonnumre
    • Kontonumre og organisasjonsnumre
    • API-nøkler, tokens og passord
    • Kommersielle nøkkeltall (timepris, dagspris, prosjektsum, margin og rabatt)
    • Konfidensielle nøkkelord («konfidensielt», «hemmelig» m.fl.)
    • Klientnavn fra intern klientliste
    • Høy-entropiske strenger (mulige hemmelige nøkler)

    Trafikklysnivåer

    Grønn — Ingen sensitive funn. Dokumentet er sannsynligvis trygt.
    Gul — Potensielt sensitive funn. Se gjennom listen og vurder.
    Rød — Sensitive funn. Ikke del uten manuell gjennomgang.
    Svart — Kritiske funn (hemmeligheter, ID-numre). Del ikke.

    Hviteliste — reduserer falske alarmer

    Gule og røde funn kan legges til din personlige hviteliste ved å klikke «+ Hviteliste». Disse verdiene vises ikke i fremtidige skanninger. Over tid tilpasser verktøyet seg ditt arbeidsmønster.

    Hvitelisten lagres lokalt og deles ikke med andre.

    Anonymisering og eksport

    Etter scanning kan du velge funn og generere en anonymisert versjon. For DOCX, PPTX, XLSX og PDF erstattes teksten direkte i filen. For andre formater genereres en renset .md-fil. Funn kan også eksporteres som JSON eller CSV.

    ⚠ PDF-redaktering erstatter tekst i innholdsstrømmen, men tekst i metadata eller bilder anonymiseres ikke. Kontroller resultatet.

    XLENT Compliance-scanner

    Protect sensitive client data — check the document before pasting into ChatGPT, Claude, or Copilot.

    What does it do?

    Compliance-scanner automatically analyses documents and searches for sensitive information such as national ID numbers, personal names, API keys, commercially sensitive figures (hourly rates, daily rates, project totals, margins, and discounts), and confidential keywords. Findings are shown as a traffic-light risk level.

    🔒100% local processing — no documents, no text, and no findings are ever sent over the internet.
    The file never leaves your machine.

    What does the tool look for?

    • National ID numbers and birth dates (Norwegian mod-11 validation)
    • Personal names — via machine learning (spaCy NER, downloaded automatically)
    • Email addresses and phone numbers
    • Bank account numbers and organisation numbers
    • API keys, tokens, and passwords
    • Commercial metrics (hourly rate, daily rate, project total, margin, and discount)
    • Confidential keywords ("confidential", "secret", etc.)
    • Client names from the internal client list
    • High-entropy strings (possible secret keys)

    Traffic light levels

    Green — No sensitive findings. The document is likely safe.
    Yellow — Potentially sensitive findings. Review and decide.
    Red — Sensitive findings. Do not share without manual review.
    Black — Critical findings (secrets, ID numbers). Do not share.

    Whitelist — reduces false positives

    Yellow and red findings can be added to your personal whitelist by clicking "+ Whitelist". Those values will not appear in future scans.

    The whitelist is stored locally and is never shared.

    Anonymisation and export

    After scanning you can select findings and generate an anonymised version. For DOCX, PPTX, XLSX, and PDF the text is replaced directly in the file. Findings can also be exported as JSON or CSV.

    ⚠ PDF redaction replaces text in the content stream, but text in metadata or images is not anonymised. Verify the result.
