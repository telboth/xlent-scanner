      XLENT Compliance-scanner

      Beskytt sensitive kundedata — sjekk dokumentet før du limer inn i ChatGPT, Claude eller Copilot.

      Hva gjør verktøyet?

      Compliance-scanner analyserer dokumenter automatisk og leter etter sensitiv informasjon som personnumre, navn, API-nøkler, kommersielt sensitive tall (timespriser, dagspriser, prosjektsummer, marginer og rabatter) og konfidensielle nøkkelord. Funnene presenteres som et trafikklysnivå slik at du raskt ser om dokumentet er trygt å dele.

      🔒100 % lokal kjøring — ingen dokumenter, tekst eller funn sendes over internett.
      Filen forlater aldri maskinen din.

      Hva søker verktøyet etter?

      • Personnumre og fødselsdatoer (norsk mod-11-validering)
      • Personnavn — via maskinlæring (spaCy NER, lastes ned via Innstillinger → Navnemodeller)
      • E-postadresser og telefonnumre
      • Bankdetaljer (kontonumre, IBAN og SWIFT/BIC) og organisasjonsnumre
      • API-nøkler, tokens og passord
      • Kommersielle nøkkeltall (timepris, dagspris, prosjektsum, margin og rabatt)
      • Konfidensielle nøkkelord («konfidensielt», «hemmelig» m.fl.)
      • Firmanavn fra intern firmaliste
      • Høy-entropiske strenger (mulige hemmelige nøkler)
      • Medisinske opplysninger som sykdommer, diagnoser og medisiner når AI-dybdeskann er valgt
      • Egendefinerte blacklist- og regex-mønstre fra Innstillinger

      Trafikklysnivåer

      Grønn — Ingen sensitive funn. Dokumentet er sannsynligvis trygt.
      Gul — Potensielt sensitive funn. Se gjennom listen og vurder.
      Rød — Sensitive funn. Ikke del uten manuell gjennomgang.
      Svart — Kritiske funn (hemmeligheter, ID-numre). Del ikke.

      Hviteliste — reduserer falske alarmer

      Gule og røde funn kan legges til din personlige hviteliste ved å klikke «+ Hviteliste». Disse verdiene vises ikke i fremtidige skanninger. Over tid tilpasser verktøyet seg ditt arbeidsmønster.

      Hvitelisten lagres lokalt og deles ikke med andre.

      Anonymisering og eksport

      Etter scanning kan du velge hvilke funn som skal brukes og generere en redigert/redacted versjon. For DOCX, PPTX, XLSX og PDF erstattes teksten direkte i filen. For andre formater genereres en renset .md-fil. Kommentarer, speaker notes og støttede annotasjoner kan fjernes, og resultatet kan re-scannes manuelt med egen knapp. Funn kan eksporteres som JSON/CSV, full HTML/PDF-rapport eller batch-/auditrapport for mappeskann.

      ⚠ PDF-redaktering erstatter tekst i innholdsstrømmen, men tekst i metadata eller bilder anonymiseres ikke. Kontroller resultatet.

      Arbeidsflyter og overvåking

      • Fil, lim inn og mappe: Skann enkeltfiler, flere filer via drag-and-drop, tekst fra utklippstavlen eller hele mapper. Mappeskann kan kjøres rekursivt med maks antall filer, maks dybde, tabellvisning, tooltip per fil, full rapport ved klikk, filter, sortering, eksport og batch-redaction av valgte filer.
      • Mappevakt: Opptil tre valgte mapper, for eksempel Downloads, overvåkes lokalt. Nye og endrede filer skannes når de er ferdig skrevet, og resultatene vises i historikken og bakgrunnsstatusen.
      • Utklippstavle-vakt: Varsler lokalt hvis du kopierer rødt eller svart innhold før det limes inn i et annet verktøy. Innholdet lagres ikke.
      • OCR for bilde-PDF: Hvis et dokument ser ut til å inneholde lite eller ingen tekst, kan du kjøre en OCR-skann for å hente tekst fra innskannede sider.
      • Desktop, web og hurtigvalg: Programmet kan kjøres som desktop-app eller lokal webmodus. Windows høyreklikk og macOS Finder Quick Action kan sende filer direkte til scanneren når installasjonsskriptet er brukt.

      Tilpasning og integrasjoner

      • Personlig hviteliste, blacklist og ignore.toml kan redigeres i Innstillinger.
      • Egendefinerte regex-mønstre kan legges inn via UI for interne saksnumre, prosjektkoder eller andre organisasjonsspesifikke formater.
      • Redaction-profiler kan brukes for å velge mellom normal, streng og mer konservativ scanning.
      • Microsoft 365-tags og sensitivity/retention-labels kan leses, foreslås og skrives der Graph-tilgang er konfigurert, inkludert lokal mapping fra trafikklysnivå til ønsket label.
      • Et lokalt API med Swagger/OpenAPI-side gjør det mulig å bygge andre frontender, for eksempel Power Apps, uten å endre desktop-appen.
      • Appen kan sjekke GitHub for ny versjon, laste ned riktig installer for operativsystemet og vise installasjonsskript der dette er relevant.

      Dybdeskann med AI (Ollama)

      Dybdeskann-fanen sender dokumentteksten til en lokal AI-modell via Ollama og ber den identifisere sensitiv informasjon som regel-baserte detektorer kan misse – for eksempel navn i uvanlige formater, adresser uten postnummer, eller budsjettall i fritekst.

      Du velger selv hvilke kategorier AI-en skal lete etter: navn, adresser, telefonnumre, personnumre, bankdetaljer, firmanavn, budsjetttall og valgfritt medisinske opplysninger. AI-funn flettes inn i den vanlige funnlisten og kan brukes i samme redaction-flyt som regelbaserte funn.

      ⏱️Maskinvarekrav: Dybdeskanning kan ta fra noen sekunder til flere minutter avhengig av dokumentstørrelse. Anbefalt maskin: minst 16 GB RAM og en relativt moderne CPU (fra ca. 2018 eller nyere). GPU-akselerasjon (NVIDIA/AMD) benyttes automatisk dersom det er tilgjengelig og gir betydelig kortere analysetid. Programmet viser om det kjører på CPU eller GPU.

      Ollama lastes ned gratis fra ollama.ai. Anbefalt modell: llama3.2:3b (~1,9 GB).

      What does it do?

      Compliance-scanner automatically analyses documents and searches for sensitive information such as national ID numbers, personal names, API keys, commercially sensitive figures (hourly rates, daily rates, project totals, margins, and discounts), and confidential keywords. Findings are shown as a traffic-light risk level.

      🔒100% local processing — no documents, no text, and no findings are ever sent over the internet.
      The file never leaves your machine.

      What does the tool look for?

      • National ID numbers and birth dates (Norwegian mod-11 validation)
      • Personal names — via machine learning (spaCy NER, download via Settings → Name models)
      • Email addresses and phone numbers
      • Bank details (account numbers, IBAN, and SWIFT/BIC) and organisation numbers
      • API keys, tokens, and passwords
      • Commercial metrics (hourly rate, daily rate, project total, margin, and discount)
      • Confidential keywords ("confidential", "secret", etc.)
      • Company names from the internal company list
      • High-entropy strings (possible secret keys)
      • Medical information such as diseases, diagnoses, and medicines when AI Deep Scan is enabled
      • Custom blacklist and regex patterns from Settings

      Traffic light levels

      Green — No sensitive findings. The document is likely safe.
      Yellow — Potentially sensitive findings. Review and decide.
      Red — Sensitive findings. Do not share without manual review.
      Black — Critical findings (secrets, ID numbers). Do not share.

      Whitelist — reduces false positives

      Yellow and red findings can be added to your personal whitelist by clicking "+ Whitelist". Those values will not appear in future scans.

      The whitelist is stored locally and is never shared.

      Redaction and export

      After scanning you can select which findings to use and generate a redacted version. For DOCX, PPTX, XLSX, and PDF the text is replaced directly in the file. Comments, speaker notes, and supported annotations can be removed, and the result can be manually re-scanned with a dedicated button. Findings can be exported as JSON/CSV, full HTML/PDF reports, or batch/audit reports for folder scans.

      ⚠ PDF redaction replaces text in the content stream, but text in metadata or images is not redacted. Verify the result.

      Workflows and monitoring

      • File, paste, and folder: Scan single files, multiple files via drag-and-drop, pasted text, or whole folders. Folder scans can run recursively with maximum file count, maximum depth, table view, per-file tooltips, full report on click, filtering, sorting, export, and batch redaction of selected files.
      • Folder watch: Up to three selected folders, for example Downloads, are monitored locally. New and changed files are scanned once they are stable, and results appear in history and the background status panel.
      • Clipboard guard: Warns locally if you copy red or black content before pasting it into another tool. Clipboard content is not stored.
      • OCR for image PDFs: If a document appears to contain little or no text, you can run an OCR scan to extract text from scanned pages.
      • Desktop, web, and quick actions: The app can run as a desktop app or in local web mode. Windows right-click and macOS Finder Quick Action can send files directly to the scanner when the installer script has been used.

      Customisation and integrations

      • Personal whitelist, blacklist, and ignore.toml can be edited in Settings.
      • Custom regex patterns can be added through the UI for internal case numbers, project codes, or other organisation-specific formats.
      • Redaction profiles can be used to choose between normal, strict, and more conservative scanning.
      • Microsoft 365 tags and sensitivity/retention labels can be read, suggested, and written when Graph access is configured, including local mapping from traffic-light level to target label.
      • A local API with a Swagger/OpenAPI page makes it possible to build other frontends, such as Power Apps, without changing the desktop app.
      • The app can check GitHub for new versions, download the correct installer for the operating system, and show installer scripts where relevant.

      The Deep Scan tab sends document text to a local AI model via Ollama and asks it to identify sensitive information that rule-based detectors may miss — for example names in unusual formats, addresses without postal codes, or budget figures embedded in free text.

AI findings are merged into the normal findings list and can be used in the same redaction flow as rule-based findings.

The app shows whether it is running on CPU or GPU.

      Après le scan, vous pouvez choisir les résultats à utiliser et générer une version expurgée/redacted. Pour DOCX, PPTX, XLSX et PDF, le texte est remplacé directement dans le fichier. Les commentaires, notes de présentation et annotations prises en charge peuvent être supprimés, et le résultat peut être rescanné manuellement avec un bouton dédié. Les résultats peuvent être exportés en JSON/CSV, rapport HTML/PDF complet ou rapport batch/audit pour les analyses de dossier.

      ⚠ Dans les PDF, le texte est remplacé dans le flux de contenu, mais le texte des métadonnées ou des images n’est pas anonymisé. Vérifiez le résultat.

      Flux de travail et surveillance

      • Fichier, collage et dossier : analysez des fichiers seuls, plusieurs fichiers par glisser-déposer, du texte collé ou des dossiers entiers. L’analyse de dossier peut être récursive avec un nombre maximal de fichiers, une profondeur maximale, une vue tableau, des infobulles par fichier, un rapport complet au clic, filtres, tri, export et redaction batch des fichiers sélectionnés.
      • Surveillance de dossier : jusqu’à trois dossiers choisis, par exemple Téléchargements, sont surveillés localement. Les fichiers nouveaux ou modifiés sont analysés lorsqu’ils sont stables, puis affichés dans l’historique et le statut d’arrière-plan.
      • Surveillance du presse-papiers : avertit localement si vous copiez du contenu rouge ou noir avant de le coller dans un autre outil. Le contenu n’est pas stocké.
      • OCR pour PDF image : si un document contient peu ou pas de texte, vous pouvez lancer un OCR pour extraire le texte des pages scannées.
      • Desktop, web et actions rapides : l’application peut fonctionner comme application desktop ou en mode web local. Le clic droit Windows et Finder Quick Action sur macOS peuvent envoyer des fichiers directement au scanner lorsque le script d’installation a été utilisé.

      Personnalisation et intégrations

      • La liste blanche personnelle, la blacklist et ignore.toml peuvent être modifiées dans les paramètres.
      • Des expressions régulières personnalisées (regex) peuvent être ajoutées via l’interface pour des numéros de dossier internes, codes projet ou autres formats propres à l’organisation.
      • Les profils de redaction permettent de choisir entre une analyse normale, stricte ou plus conservatrice.
      • Les tags Microsoft 365 et les labels de sensibilité/rétention peuvent être lus, proposés et écrits lorsque l’accès Graph est configuré, avec mapping local du niveau de risque vers le label cible.
      • Une API locale avec page Swagger/OpenAPI permet de créer d’autres frontends, par exemple Power Apps, sans modifier l’application desktop.
      • L’application peut vérifier GitHub pour les nouvelles versions, télécharger l’installer adapté au système d’exploitation et afficher les scripts d’installation pertinents.

      Analyse IA approfondie (Ollama)

      L’onglet d’analyse approfondie envoie le texte du document à un modèle IA local via Ollama pour identifier des informations sensibles que les détecteurs à règles peuvent manquer — par exemple des noms dans des formats inhabituels, des adresses sans code postal ou des montants de budget en texte libre.

L’accélération GPU (NVIDIA/AMD) est utilisée automatiquement si disponible et réduit nettement le temps d’analyse. L’application indique si elle tourne sur CPU ou GPU.

      XLENT Compliance-scanner

      Proteja los datos sensibles de clientes — revise el documento antes de pegarlo en ChatGPT, Claude o Copilot.

      ¿Qué hace la herramienta?

      El escáner de cumplimiento analiza documentos automáticamente y busca información sensible como números de identificación personal, nombres, claves API, métricas comerciales sensibles (tarifa por hora, tarifa diaria, total de proyecto, margen y descuento) y palabras clave confidenciales. Los hallazgos se muestran como nivel de riesgo tipo semáforo.

      🔒Ejecución 100 % local — no se envían documentos, texto ni hallazgos por Internet.
      El archivo nunca sale de su equipo.

      ¿Qué busca la herramienta?

      • Números de identificación y fechas de nacimiento (validación noruega Mod-11)
      • Nombres de personas — mediante aprendizaje automático (spaCy NER, descarga en Configuración → Modelos de nombres)
      • Direcciones de correo y números de teléfono
      • Datos bancarios (números de cuenta, IBAN y SWIFT/BIC) y números de organización
      • Claves API, tokens y contraseñas
      • Métricas comerciales (tarifa por hora, tarifa diaria, total del proyecto, margen y descuento)
      • Palabras clave confidenciales ("confidencial", "secreto", etc.)
      • Nombres de empresas de la lista interna
      • Cadenas de alta entropía (posibles claves secretas)
      • Información médica como enfermedades, diagnósticos y medicamentos cuando se activa el escaneo profundo con IA
      • Blacklist y patrones regex personalizados desde Configuración

      Niveles de semáforo

      Verde — No hay hallazgos sensibles. El documento probablemente es seguro.
      Amarillo — Hallazgos potencialmente sensibles. Revise y evalúe.
      Rojo — Hallazgos sensibles. No compartir sin revisión manual.
      Negro — Hallazgos críticos (secretos, identificadores). No compartir.

      Lista blanca — reduce falsos positivos

      Los hallazgos amarillos y rojos pueden añadirse a su lista blanca personal con "+ Lista blanca". Esos valores no aparecerán en análisis futuros.

      La lista blanca se guarda localmente y nunca se comparte.

      Anonimización y exportación

      Después del análisis puede elegir qué hallazgos usar y generar una versión redactada/redacted. En DOCX, PPTX, XLSX y PDF el texto se reemplaza directamente en el archivo. Se pueden eliminar comentarios, notas del orador y anotaciones compatibles, y el resultado puede volver a escanearse manualmente con un botón dedicado. También puede exportar hallazgos como JSON/CSV, informe HTML/PDF completo o informe batch/audit para escaneos de carpeta.

      ⚠ En PDF se reemplaza texto en el flujo de contenido, pero el texto en metadatos o imágenes no se anonimiza. Verifique el resultado.

      Flujos de trabajo y supervisión

      • Archivo, pegar y carpeta: escanee archivos individuales, varios archivos mediante arrastrar y soltar, texto pegado o carpetas completas. El escaneo de carpetas puede ejecutarse recursivamente con número máximo de archivos, profundidad máxima, vista de tabla, tooltip por archivo, informe completo al hacer clic, filtros, ordenación, exportación y redaction batch de archivos seleccionados.
      • Vigilancia de carpeta: hasta tres carpetas elegidas, por ejemplo Descargas, se supervisan localmente. Los archivos nuevos o modificados se escanean cuando están estables y los resultados aparecen en el historial y el estado de fondo.
      • Vigilancia del portapapeles: avisa localmente si copia contenido rojo o negro antes de pegarlo en otra herramienta. El contenido no se guarda.
      • OCR para PDF de imagen: si un documento contiene poco o ningún texto, puede ejecutar OCR para extraer texto de páginas escaneadas.
      • Desktop, web y acciones rápidas: la aplicación puede ejecutarse como app de escritorio o en modo web local. El clic derecho de Windows y Finder Quick Action en macOS pueden enviar archivos directamente al scanner cuando se ha usado el script de instalación.

      Personalización e integraciones

      • La lista blanca personal, la blacklist e ignore.toml se pueden editar en Configuración.
      • Se pueden añadir patrones regex personalizados desde la UI para números de caso internos, códigos de proyecto u otros formatos específicos de la organización.
      • Los perfiles de redaction permiten elegir entre análisis normal, estricto o más conservador.
      • Las etiquetas Microsoft 365 y los labels de sensibilidad/retención pueden leerse, sugerirse y escribirse cuando Graph está configurado, con mapping local del nivel de semáforo al label objetivo.
      • Una API local con página Swagger/OpenAPI permite crear otros frontends, por ejemplo Power Apps, sin cambiar la aplicación desktop.
      • La app puede comprobar GitHub para nuevas versiones, descargar el instalador correcto para el sistema operativo y mostrar scripts de instalación cuando corresponda.

      Escaneo profundo con IA (Ollama)

      La pestaña de escaneo profundo envía el texto del documento a un modelo de IA local vía Ollama para identificar información sensible que los detectores basados en reglas pueden pasar por alto — por ejemplo nombres en formatos inusuales, direcciones sin código postal o cifras de presupuesto en texto libre.

      Usted elige las categorías que la IA debe buscar: nombres, direcciones, teléfonos, identificaciones personales, datos bancarios, nombres de empresa, montos de presupuesto e información médica opcional. Los hallazgos de IA se fusionan con la lista normal y pueden usarse en el mismo flujo de redaction que los hallazgos basados en reglas.

      ⏱️Requisitos de hardware: el escaneo profundo puede tardar desde segundos hasta varios minutos según el tamaño del documento. Recomendado: al menos 16 GB de RAM y una CPU relativamente moderna (aprox. 2018 o posterior). La aceleración por GPU (NVIDIA/AMD) se usa automáticamente cuando está disponible y reduce significativamente el tiempo de análisis. La app muestra si se ejecuta en CPU o GPU.

      Ollama se descarga gratis en ollama.ai. Modelo recomendado: llama3.2:3b (~1,9 GB).
